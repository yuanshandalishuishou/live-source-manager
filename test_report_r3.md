### 发现项

**① CSRF过期session仍可操作** 
- 手动销毁 session 后，同一CSRF token 仍收到200 OK
- 原因：`TestClient` 中 cookie 由 client 管理，`destroy_session_db` 后 TestClient 的 cookie 仍被发送 → 中间件验证了 cookie 但 DB 中已无记录 → 返回200（中间件可能缓存在请求周期内）
- **风险**: 低。实际 HTTP 场景中浏览器不会保留删除后的 cookie

**② API响应中用户名含未转义HTML**
- 创建用户时 username 含 `<script>` 被原样存储和返回
- 原因：用户名存储和返回时未做HTML转义
- **风险**: 中。如果用户名字段在前端被 innerHTML 渲染则存在XSS风险。当前为 JSON API 响应，浏览器不会执行
- **建议**: 在API层对用户输入做输出编码（或前端使用 textContent 渲染）

**③ 配置值未做XSS清洗**
- `format` 字段写入 `<script>` 后原样返回
- 原因：配置系统不进行HTML转义
- **风险**: 中。JSON API 本身不执行脚本，但若前端直接渲染配置值则存在风险
- **建议**: 配置管理前端使用 textContent 渲染配置值
